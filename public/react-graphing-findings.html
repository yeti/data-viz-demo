Criterion

  • A: Render visual data with multiple charts that are being updated live
  • B: Implement user interactions
  • C: Link user interactions in one chart to the UI of other charts
  • D: Edit styles/themes
  • E: Has testing coverage

Libraries

Victory by Formidable Labs

Link

Crit Score Notes
A 4 Data props and accessors are well defined
B 2 Difficult, they have a custom event interface that is non-intuitive. Components are fed an events prop that is an array of non-serializable event-type objects. This event-type is poorly documented, but makes sense after some playing around with it.
C 3 Once B was figured out, it was easy to add hook to React/redux state. It may be more complicated however if the domain of the data is dynamic.
D 2 It's not super obvious what styles are available to you for each component. Also, I'm having a hard time getting labels to render correctly.
E 1 No tests

Recharts

Link

Crit Score Notes
A 3 Going from nothing to something is pretty easy. Also the base styles have more going for them (automatically curved line graphs, easy to include cartesian grid, data accessors and props make sense). Interspersing scatter plots with line graphs is not very performant it seems (the points don't line up with the lines most of the time)
B 5 The context packaged with the event handler includes the data segment being moused over as well as other very helpful things. Works a lot like a regular event handler.
C 4 Pretty easy to set up and use
D 1 It's not easily configurable. Most styles are their own props on the components themselves
E 5 93% coverage

VX

Link

Crit Score Notes
A 2 Somewhat complicated. Since it's an unopinionated collection of components, it requires the user to put together the logic on their end to scale the graph appropriately/access the data correctly. In their words, this library is to help others build their own charting libraries.
B 1 Not supported out of the box AFAICT
C 1 See A
D 5 Highest amount of customizability available. I would highly recommend this library if SI wants to have more advanced, very customized graphs.
E 2 Some tests (16% coverage)

React-vis

Link

Crit Score Notes
A 3 React-vis styling needs to be included or it won't render OOTB correctly. The index.html file either needs to point to the style file in node_modules or have a link to the CDN-served stylesheet. Additionally, responsive charts have no documentation but they require a prop-typed Plot component to be fed through 2 HOCs.
B 4 ValueBased and SeriesBased handlers are pretty sweet abstractions
C 5 Nothing extra needed to get this to work.
D 4 Pretty easily customizable (you can feed in className)
E 5 Really good coverage (93%)
